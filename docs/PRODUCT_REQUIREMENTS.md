# 产品功能文档（V1.1）

## 1. 产品概述
本 App 是一个面向游戏开黑等场景的多人语音聊天产品。用户通过 Google 或 Apple 账号登录后，可创建和管理自己的频道，搜索并关注其他用户，在首页查看已关注用户的频道，并从频道列表进入多人语音聊天页面进行实时语音沟通。

## 2. 目标与范围
### 2.1 产品目标
1. 降低注册与登录门槛，支持 Google 登录和 Apple 登录快速进入产品。
2. 以”用户频道”为核心组织内容，实现关注与内容订阅。
3. 提供低延迟、稳定的多人语音聊天能力，满足游戏组队沟通需求。
4. 提供清晰的个人主页信息管理能力，支持基础资料和频道名称编辑。

### 2.2 版本范围（本次实现）
1. Google 登录 + Apple 登录。
2. 一人一频道（可自定义频道名称）。
3. 用户搜索与关注。
4. 首页展示已关注用户的频道，并支持进入语音聊天页。
5. 多人语音聊天页面（加入频道语音房间、查看在线成员、静音/离开）。
6. 个人主页展示与编辑（基本信息、频道名称）。
7. 账号登出（token 失效后自动登出，跳转登录页）。

### 2.3 非目标（本版本不包含）
1. 多频道体系（每个用户仅 1 个频道）。
2. 复杂推荐算法。
3. 私信、评论、点赞等社交互动。
4. 管理后台与运营工具。

## 3. 用户与核心场景
### 3.1 目标用户
1. 希望关注特定创作者或用户频道的普通用户。
2. 希望维护个人频道身份信息的内容发布者。

### 3.2 核心场景
1. 新用户首次登录（Google 或 Apple）并自动创建频道。
2. 用户在首页点击”搜索用户”入口，搜索某人并点击关注。
3. 用户在首页通过顶部固定入口进入”我的频道”。
4. 用户在首页查看已关注频道列表。
5. 用户从首页点击某个频道进入语音聊天页面，与多人实时语音沟通。
6. 用户进入个人主页修改昵称、头像、频道名称。
7. 用户 token 失效后自动登出，重新登录。

## 4. 功能需求
### 4.1 登录（Google + Apple）
#### 功能说明
用户通过 Google 账号或 Apple ID 完成登录，首次登录自动创建账号与默认频道。登录页同时展示两个登录入口。

#### 关键规则
1. 支持 Google Sign-In 和 Apple Sign-In 两种登录方式。
2. 首次登录时创建用户记录与默认频道。
3. 非首次登录直接进入首页（token 持久化，免登录）。
4. 登录态需持久化，存储 access_token 至 UserDefaults，下次启动自动恢复。
5. Apple Sign-In 支持"隐藏我的电子邮件"中继地址，App 可正常使用。
6. Apple Sign-In 首次登录携带姓名与邮箱；后续登录不再下发，由本地缓存补充。

#### 关键字段（Google）
1. Google 唯一标识（id_token → 服务端校验 sub）。
2. 用户昵称（name）。
3. 用户头像（avatar_url）。

#### 关键字段（Apple）
1. Apple Identity Token（id_token → 服务端校验 sub）。
2. 用户姓名（fullName，仅首次登录下发）。
3. 用户邮箱（email，仅首次登录下发，可为 Hide My Email 中继地址）。

### 4.1.1 账号登出
#### 功能说明
当服务端返回 401 Unauthorized 时，客户端自动清除本地 token 与用户信息，并跳转至登录页，要求用户重新认证。

#### 关键规则
1. 任意 API 请求返回 401 时触发登出流程。
2. 清除 UserDefaults 中的 access_token、user_id、user_name、user_email、user_avatar_url。
3. 跳转至登录页，之前的导航栈全部销毁。
4. 登出为被动触发（token 过期/失效），当前版本无主动登出入口。

### 4.2 用户频道（1:1）
#### 功能说明
每位用户拥有一个唯一频道，频道名称支持用户自定义。

#### 关键规则
1. 一个用户只能对应一个频道。
2. 频道名称支持修改。
3. 频道名称需校验长度与非法字符。
4. 频道名称修改后应实时反映在首页与个人主页。

#### 建议校验规则
1. 长度：2-30 字符。
2. 不允许纯空格。
3. 禁止敏感词与系统保留词（如 admin、official 等）。

### 4.3 搜索用户与关注
#### 功能说明
用户可通过关键词搜索其他用户，并执行关注操作。

#### 搜索规则
1. 支持按昵称、频道名称模糊搜索。
2. 搜索结果展示：头像、昵称、频道名称、关注状态。
3. 不展示自己作为可关注对象。

#### 关注规则
1. 支持关注与取消关注。
2. 关注关系唯一，不可重复创建。
3. 关注成功后，目标频道进入首页列表。
4. 取消关注后，目标频道从首页列表移除。

### 4.4 首页（关注频道聚合）
#### 功能说明
首页用于展示当前用户已关注用户的频道列表，并作为进入语音聊天页面的主入口。首页顶部提供固定入口“我的频道”和“搜索用户”入口，分别用于快速进入当前用户自己的频道语音房间和用户搜索页面。

#### 展示规则
1. 按最近活跃时间或关注时间排序（建议最近活跃优先）。
2. 顶部固定展示“我的频道”入口卡片与“搜索用户”入口按钮，不参与关注列表排序。
3. 每个频道卡片展示：频道名称、用户昵称、头像、最近更新时间（若有）。
4. 无关注数据时仍保留“我的频道”与“搜索用户”入口，并展示空态引导去“搜索用户”。

#### 交互规则
1. 点击顶部“我的频道”入口，进入当前用户自己的频道语音聊天页面。
2. 点击顶部“搜索用户”入口，进入搜索页进行用户检索与关注操作。
3. 点击关注频道卡片进入对应频道的多人语音聊天页面。
4. 支持下拉刷新或重新加载（按客户端能力实现）。

### 4.5 多人语音聊天页面
#### 功能说明
用户从首页顶部“我的频道”入口或关注频道列表进入语音聊天页面后，可在该频道与多人进行实时语音沟通。

#### 页面信息
1. 频道名称与房间状态。
2. 在线成员列表（头像、昵称、麦克风状态）。
3. 语音控制区（麦克风开关、扬声器或耳机模式、离开房间）。

#### 关键规则
1. 进入页面即尝试加入对应频道语音房间。
2. 同一频道支持多人同时在线语音。
3. 用户离开页面或点击离开按钮时退出语音房间。
4. 麦克风默认关闭或按系统策略执行，首次使用需处理麦克风权限。
5. 语音连接异常时需提示重连状态与失败原因。

### 4.6 个人主页（展示与编辑）
#### 功能说明
个人主页展示当前用户基础信息与频道名称，支持编辑并保存。

#### 展示内容
1. 头像。
2. 昵称。
3. 邮箱（可选展示）。
4. 频道名称。

#### 编辑能力
1. 支持编辑昵称与频道名称（头像编辑可选）。
2. 保存前进行字段校验。
3. 保存成功后提示并回显最新信息。
4. 保存失败时提示错误原因并保留用户输入。

## 5. 页面与信息架构
1. 登录页：Google 登录入口。
2. 首页：顶部固定“我的频道”入口 + “搜索用户”入口 + 已关注频道列表。
3. 多人语音聊天页：从首页频道列表点击进入。
4. 搜索页：用户搜索与关注操作。
5. 个人主页：资料展示与编辑。

## 6. 核心业务流程
### 6.1 首次登录流程
1. 用户点击 Google 登录。
2. 鉴权成功返回用户身份信息。
3. 系统创建用户。
4. 系统创建该用户唯一频道（默认频道名可由昵称生成）。
5. 跳转首页。

### 6.2 搜索并关注流程
1. 用户在首页点击“搜索用户”入口进入搜索页。
2. 用户输入关键词搜索。
3. 系统返回匹配用户列表。
4. 用户点击关注。
5. 系统创建关注关系。
6. 返回首页后刷新并展示新关注频道。

### 6.3 编辑个人主页流程
1. 用户进入个人主页点击编辑。
2. 修改昵称和频道名称。
3. 点击保存并校验。
4. 系统更新数据。
5. 页面提示成功并刷新展示。

### 6.4 进入语音聊天流程
1. 用户在首页点击顶部“我的频道”入口或某个已关注频道。
2. 系统打开对应频道语音聊天页面并发起入房。
3. 入房成功后展示在线成员与语音控制区。
4. 用户可进行静音或取消静音，并持续语音沟通。
5. 用户点击离开或返回时退出语音房间。

## 7. 数据模型（简版）
### 7.1 User
1. user_id
2. google_sub（唯一）
3. name
4. avatar_url
5. email
6. created_at
7. updated_at

### 7.2 Channel
1. channel_id
2. owner_user_id（唯一）
3. channel_name
4. created_at
5. updated_at

### 7.3 Follow
1. follow_id
2. follower_user_id
3. followee_user_id
4. created_at
5. 唯一索引：follower_user_id + followee_user_id

## 8. 权限与安全要求
1. 所有用户数据接口需校验登录态（Bearer token）。
2. 只能编辑自己的个人信息与频道名称。
3. 关注关系操作仅对当前登录用户生效。
4. Google id_token 和 Apple identity token 需服务端校验签名与有效期。
5. 避免在客户端暴露敏感密钥。
6. 服务端返回 401 时客户端立即清除本地 token 并触发自动登出。

## 9. 异常与边界处理
1. 登录失败：界面提示具体错误信息，允许重试；用户取消 Apple 授权时静默忽略。
2. 搜索无结果：展示空态。
3. 重复关注：接口幂等返回成功或提示已关注。
4. 自关注：禁止并提示。
5. 频道名称不合法：前后端双重校验并提示具体原因。
6. 网络错误：提示网络异常并支持重试。
7. token 失效（401）：自动登出并跳转登录页。

## 10. 验收标准（UAT）
1. 用户可通过 Google 账号成功登录并进入首页。
2. 用户可通过 Apple ID 成功登录并进入首页（支持”隐藏我的电子邮件”）。
3. 首次登录后自动生成个人频道。
4. 首页顶部固定显示”搜索用户”入口，点击后可进入搜索页。
5. 用户可搜索到他人并成功关注或取消关注。
6. 首页顶部固定显示”我的频道”入口，点击后可进入自己的频道语音聊天页面。
7. 首页展示当前用户已关注的频道，并可点击进入语音聊天页面。
8. 语音聊天页可正常加入房间、展示在线成员、执行静音与离开操作。
9. 个人主页可查看并编辑基本信息与频道名称，保存后正确生效。
10. token 失效场景下自动清除本地认证信息并跳转登录页。
11. 异常场景提示明确，不出现页面卡死或数据错乱。
